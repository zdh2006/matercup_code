## 二、第一类：组合优化与调度类

### 共性特征与适用模型

**核心问题结构：** 在有限资源或约束条件下，从离散决策空间中寻找使目标函数最优的方案。

**建模思路通用流程：**
1. 识别**决策变量**（选哪个/选多少/怎么排）
2. 刻画**约束**（容量、顺序、独占性等）
3. 构建**目标函数**（最小化成本/最大化收益/最小化时间）
4. 选择求解方法：精确算法（小规模）→ 启发式/元启发式（大规模）

**适用模型体系：**

- **整数线性规划（ILP / MILP）**：问题规模较小时的精确模型
- **QUBO模型**：决策变量为0/1二值变量的无约束二次优化，适合量子计算求解
- **贪心算法 + 局部搜索**：构造可行解后不断迭代改进
- **遗传算法 / 模拟退火 / 粒子群（GA/SA/PSO）**：大规模组合空间的近似求解
- **列生成 / 分支定价**：切割排样等具有特殊结构问题的精确方法

### 2023年A题：量子计算机在信用评分卡组合优化中的应用

**题目逻辑剖析：**

题目分三层递进：
- **问题1**：单张卡、选1个阈值 → 最简单的单变量枚举
- **问题2**：固定3张卡、优化各自阈值 → 三个独立离散变量的联合优化
- **问题3**：从100张卡中选3张并设定阈值 → 选卡+选阈值的双层组合优化

**收入计算公式是核心：**
```
最终收入 = 贷款资金 × 利息率 × (∏通过率) × (1 - 平均坏账率)
         - 贷款资金 × (∏通过率) × (平均坏账率)
```

**建模思路产生过程：**

1. 目标函数是通过率（连乘）和坏账率（均值）的非线性函数 → 直接优化较难
2. 每张卡仅选1个阈值 → 引入0-1变量 $x_{i,j}$（第 $i$ 张卡选第 $j$ 个阈值），并有约束 $\sum_j x_{i,j} = 1$
3. 将乘积形式线性化（取对数）或展开为二次项 → 形成QUBO格式
4. 约束条件"每卡选且仅选一个阈值"通过惩罚项引入目标函数：
   $\min f(x) + \lambda \sum_i (\sum_j x_{i,j} - 1)^2$

**QUBO转化关键步骤：**
- 决策变量：$x_{i,j} \in \{0,1\}$，共 100×10 = 1000 个变量
- 将目标函数与约束惩罚项合并为纯二次型 $x^T Q x$
- 问题3因同时选卡和选阈值，变量数更多，需引入选卡变量 $y_i$ 与条件约束

**求解方法：** D-Wave量子退火 / SimulatedAnnealing / 枚举（小规模验证）

### 2022年B题：无人仓的搬运机器人调度问题

**题目逻辑剖析：**

三个问题形成"调度→区域划分→防碰撞"的层次结构：
- **问题1**：不考虑碰撞，最优任务分配 + 路径规划
- **问题2**：对仓库进行区域划分，实现负载均衡
- **问题3**：加入碰撞约束，设计防碰撞策略

**建模思路产生过程：**

问题1是经典的**指派问题（Assignment Problem）** + **最短路（BFS on grid graph）**的组合：
- 每个AGV分配若干出库/回库任务 → 多任务指派
- AGV在网格图上行走 → BFS/Dijkstra求最短路
- 目标：各AGV尽量忙 + 总行走路径最小 → 多目标，可加权化为单目标

问题2是**K-Medoids / Voronoi分区**思路：
- 以拣选工位为"中心"，划分储位到最近工位
- 约束：各工位分到的货物总量尽量均衡 → 带平衡约束的聚类

问题3引入**时空约束**：
- 同一节点在同一时刻只能被一个AGV占用
- 死锁检测：两AGV互相等待 → 资源等待图中出现环
- 策略：优先级抢占 + 路径重规划（A* with time dimension）

### 2021年D题：钢材制造业中的钢材切割下料问题

**题目逻辑剖析：**

典型的**切割排样（Cutting Stock Problem）**，是运筹学的经典NP难问题。题目分四问递进增加约束：
- **问题1**：只考虑5种卷料，最少原材料数量
- **问题2**：15种订单全覆盖，最少材料+最大成材率
- **问题3**：在问题2基础上加入最少换刀次数和小机器切割次数
- **问题4**：引入长度浮动比例

**建模思路产生过程：**

核心约束：圆盘剪每次最多5刀（即每组最多切出4条子条）+ 切头剪"一刀切"要求。

1. 枚举所有可行切割模式（Pattern），每种模式对应一组宽度组合
2. 以各模式使用次数为变量 → 整数线性规划（ILP）
3. 目标：最小化使用的原材料张数
4. 约束：每种订单需求量恰好满足 + 每种原材料不超库存
5. 成材率最大化作为第二目标，可通过约束松弛或字典序优化处理
6. 问题3换刀成本：相邻组的切割模式不同即为换刀，引入排序变量

**关键技巧：** 用"列生成法"动态生成切割模式，避免枚举爆炸

### 2020年C题：仓内拣货优化问题

**题目逻辑剖析：**

六问逐步深化，从单人单任务到多人多任务多复核台：
- **问题1**：计算仓库中所有元素间最短距离矩阵
- **问题2**：单任务最优拣货路线规划
- **问题3**：单人多任务，规划任务顺序和拣货路线
- **问题4**：多人多任务多复核台，全局调度优化
- **问题5**：增加一个复核台的效益评估
- **问题6**：商品摆放的建议（开放性问题）

**建模思路产生过程：**

问题1：仓库是矩形网格，存在货架障碍物 → 对每对元素用BFS/曼哈顿距离+绕障修正求最短路。

问题2：在已有距离矩阵基础上，拣货员访问一组货格 → **旅行商问题（TSP）的变体**：
- 规模较小（约20-30个货格），可用贪心最近邻 + 2-opt改进

问题3-4：多任务调度 → **并行机调度（Parallel Machine Scheduling）**：
- 任务分配：将任务分给人员（指派问题）
- 路线规划：每人针对分配到的任务序列做TSP
- 复核台约束：任务完成须送达复核台才能开始下一任务 → 类似Job-shop约束

**关键难点：** 复核台利用率与人员行走路径的多目标权衡

## 三、第二类：路径规划与轨迹优化类

### 共性特征与适用模型

**核心问题结构：** 在连续空间中，在运动学/动力学约束下，为移动体（车辆）规划满足安全和效率要求的轨迹。

**建模思路通用流程：**
1. 建立**车辆运动学模型**（Ackermann转向、曲率约束等）
2. 定义**状态空间**（位置、朝向、速度、曲率等）
3. 描述**约束**（碰撞避免、速度限制、曲率连续等）
4. 定义**优化目标**（最短时间、最平滑轨迹等）
5. 选择求解方法

**适用模型体系：**

- **Dubins曲线 / Reeds-Shepp曲线**：考虑最小转弯半径的最短路径
- **三次样条 / 多项式曲线**：保证曲率连续（C2连续）的轨迹插值
- **RRT* / A\* with kinematic constraints**：状态空间搜索
- **Clothoid（回旋曲线/欧拉螺线）**：曲率线性变化的平滑曲线，最适合实际驾驶
- **非线性规划（NLP）/ 最优控制（MPC）**：精确轨迹优化

---

### 2022年C题：自动泊车问题

**题目逻辑剖析：**

四问由简到繁：
- **问题1**：计算最小转弯半径、加速约束、曲率变化率限制（纯力学分析）
- **问题2**：给定目标车位（垂直/平行/倾斜三种）的完整泊车轨迹
- **问题3**：自主选择最优停车位 + 完整轨迹
- **问题4**：动态环境（车辆随机进出），实时重规划

**建模思路产生过程：**

问题1是**运动学分析**的基础：
- 最小转弯半径：$R_{min} = L / \tan(\delta_{max})$，其中 $L$ 为轴距，$\delta_{max}$ 为最大前轮转角
- 加速到最大速度的最短距离：在加加速度约束下，分"加加速→匀加速→减加速"三段分析
- 曲率变化率：行驶速度 $v$ 下方向盘转速限制 → $dk/ds \leq f(v, \omega_{max})$

问题2的核心是**泊车轨迹生成**：
- 垂直车位：倒车入库，典型的"切圆弧"策略（两段圆弧）
- 平行车位：经典"S形"泊车（前进+后退+圆弧组合）
- 倾斜车位：可简化为垂直车位的旋转变换

轨迹格式要求包含每时刻的：路径长度、朝向、速度、加速度、加加速度、角速度、角加速度。

问题3引入**停车位评价函数**：综合考虑距离当前位置的远近、行驶路径难度、车位类型。

问题4：引入**滚动时域规划（MPC）**思路，定期重新感知环境并更新路径。

### 2021年A题：自动驾驶中的车辆调头问题

**题目逻辑剖析：**

六问涵盖从简单调头到动态障碍的多个场景：
- **问题1**：宽敞场地一次调头，建立调头轨迹模型
- **问题2**：判断是否需要倒车（狭窄场地），建立判断准则
- **问题3**：存在静止障碍物时的避障调头
- **问题4**：存在斑马线的交规约束
- **问题5**：障碍物为移动状态
- **问题6**：算法求解成功率与效率的权衡

**建模思路产生过程：**

问题1：确定控制点位置（后轴中心）→ 使用**Reeds-Shepp曲线**描述包含倒车的最短路径，或用**三段Clothoid曲线**（进入弧→圆弧→退出弧）保证曲率连续。

问题2：判断是否需要倒车的关键在于最小调头宽度：
- 纯前进调头所需最小宽度 $W_{min} = f(R_{min}, L_{vehicle})$
- 若道路宽度 $W < W_{min}$，必须倒车

问题3-5逐步增加约束：
- 静止障碍物 → 在碰撞检测（OBB/AABB）框架下重规划
- 移动障碍物 → 引入时间维度，在时空（x, y, t）空间搜索无碰撞路径

问题6：求解效率与鲁棒性的权衡对应**算法选择策略**，RRT族（快但非最优）vs 优化方法（慢但可找到更好解）。

## 四、第三类：网络流与交通优化类

### 共性特征与适用模型

**核心问题结构：** 在图/网络上，在容量和时序约束下优化流量或时刻安排。

**适用模型：**
- **多商品网络流（MCNF）**：多类乘客/货物同时在网络中流动
- **多目标整数规划**：企业成本 + 服务水平的双目标
- **时空网络展开（Time-Expanded Network）**：将时刻表问题转化为静态网络流
- **帕累托前沿 / 加权和法**：处理多目标

---

### 2023年B题：城市轨道交通列车时刻表优化问题

**题目逻辑剖析：**

三问构成"方案设计 → 时刻表铺画 → 改进建议"的完整流程：
- **问题1**：确定大小交路区间及列车对数（开行方案）
- **问题2**：在开行方案基础上铺画等间隔平行运行图
- **问题3**：提出降低成本/提高服务水平的量化建议

**建模思路产生过程：**

问题1的核心是**满足断面客流的最小列车数**：
- 每个断面所需列车数 = ⌈断面客流 / 列车定员⌉
- 大交路覆盖全线，小交路覆盖高峰区段
- 决策变量：大交路对数 $n_L$，小交路区间 $[s_a, s_b]$，小交路对数 $n_S$
- 约束：每个断面发车数 ≥ 所需列车数；发车间隔在 [最小间隔, 最大间隔] 内
- 目标1（成本）：总走行公里（列车运营里程之和）+ 所需车辆数
- 目标2（服务）：乘客平均等待时间 + 平均在车时间

问题2：在既定开行方案下的**等间隔时刻表排程**：
- 大小交路按 n:1 比例交替发车
- 决策变量：首班车出发时刻、发车间隔
- 关键约束：追踪间隔（两车同向行驶的最小时间间隔）
- 停站时间正比于该站上下车客流量

问题3是开放性问题，可从以下方向量化分析：弹性时刻表、动态交路调整、候车客流建模。

## 五、第四类：物流网络与覆盖优化类

### 共性特征与适用模型

**核心问题结构：** 在带有容量约束的有向网络中，优化流量分配（物流调度）或设施选址（覆盖问题）。

**适用模型：**
- **最大流/最小费用流**：货量在网络中的调度
- **设施选址（P-median/P-center）**：基站位置优化
- **集合覆盖/最大覆盖问题**：以最少设施覆盖最多需求点
- **鲁棒优化**：应对预测不确定性

---

### 2023年C题：电商物流网络包裹应急调运与结构优化问题

**题目逻辑剖析：**

四问形成"预测 → 应急调运 → 网络重构 → 新增优化"的完整链条：
- **问题1**：预测2023年1月各线路每日货量（时序预测）
- **问题2**：DC5关停后的货量调配（静态网络优化）
- **问题3**：DC9关停后允许动态调整线路的优化（动态网络优化）
- **问题4**：评价节点/线路重要性，设计新增节点方案

**建模思路产生过程：**

问题1：典型的**时序预测问题**，货量受节假日、促销等影响 → 使用SARIMA或Prophet等带季节性的预测模型，或LSTM等深度学习方法。

问题2：DC5关停后，其相关线路的货量需重新分配：
- 建立**最小费用流模型**，以货量变化线路数、负载均衡度为目标
- 硬约束：各线路货量 ≤ 历史最大值（容量上限）
- 目标1：货量变化线路数最少（稀疏性）
- 目标2：线路负载尽量均衡（负载方差最小化）
- 若有超出容量的货量，则作为"未正常流转"记录

问题3：DC9关停+允许动态增删线路：
- 每天可新开线路（上限为已有线路最大运输能力）
- 每天的决策：哪些线路关闭？哪些线路新开？货量如何分配？
- 这是一个**动态规划/多期优化**问题，可按天独立求解（若无跨期依赖）

问题4：**网络重要性评价**可用PageRank、介数中心性、节点删除后最大流变化等度量；新增节点选址类似P-median。

### 2022年D题：移动通信网络站址规划和区域聚类问题

**题目逻辑剖析：**

三问涵盖"圆形覆盖选址 → 扇区覆盖选址 → 弱覆盖点聚类"：
- **问题1**：简单圆形覆盖，选站址使弱覆盖业务量的90%被覆盖
- **问题2**：每站3扇区（非圆形），在扇区角度也可优化的情况下求最优方案
- **问题3**：将距离≤20的弱覆盖点聚类（连通分量问题）

**建模思路产生过程：**

问题1是经典的**带权最大覆盖问题（MWCP）**：
- 决策变量：在哪些栅格点建基站（宏站/微站）
- 覆盖判断：欧氏距离 ≤ 覆盖范围
- 约束：新站之间及与现有站距离 > 10（最小间距）
- 目标：被覆盖弱覆盖点总业务量 ≥ 总量的90%，同时成本最小
- 问题可用**贪心+局部搜索**求解（每次选性价比最高的站址）

问题2：扇区覆盖是非圆形，增加了**角度决策变量**，问题变成混合整数非线性规划，难度增大，可用启发式方法。

问题3：聚类要求是**基于距离阈值的连通分量**，算法本质是图的BFS/DFS或并查集（Union-Find），时间复杂度可达 $O(n^2)$（建图）。

## 六、第五类：时间序列预测与需求预测类

### 共性特征与适用模型

**核心问题结构：** 利用历史数据，建立量化模型，预测未来某一指标的数值。

**建模思路通用流程：**
1. **数据探索**：趋势性、季节性、周期性、异常值分析
2. **特征工程**：节假日、促销等外生变量编码
3. **模型选择**：统计模型（ARIMA/ETS）或机器学习模型
4. **交叉验证**：时序数据用"前向滚动验证"而非随机分割
5. **评估指标**：MAE、MAPE、RMSE

**适用模型体系：**

| 场景 | 推荐模型 |
|------|----------|
| 数据量少，线性趋势 | ARIMA / Holt-Winters |
| 数据量中等，有多重季节性 | Prophet / TBATS |
| 数据量大，特征丰富 | LightGBM / XGBoost |
| 时序依赖性强 | LSTM / Transformer |
| 结合外部因素 | 回归树 + 时序特征 |

### 2020年B题：养老服务床位需求预测与运营模式研究

**题目逻辑剖析：**

四问构建"需求预测 → 商机分析 → 运营模式设计 → 政策建议"框架：
- **问题1**：基于人口结构预测床位需求规模
- **问题2**：从企业角度发现床位增长中的商机
- **问题3**：设计可持续的养老服务运营商业模式
- **问题4**：提炼核心数学问题，给出政策建议

**建模思路产生过程：**

问题1是**系统动力学 + 人口学**建模：
- 对中国人口进行分年龄段预测（Leslie矩阵或Logistic增长）
- 确定"需要机构养老"的老年人比例（渗透率）
- 床位需求 = 老年人口数 × 渗透率
- 该渗透率受城镇化、收入水平、家庭规模等因素影响

问题2的"商机"分析是**供需缺口分析**：
- 现有床位（供给）vs 预测需求 → 缺口即商机
- 细分维度：城市等级、养老类型（公办/民营/公建民营）

问题3涉及**收支平衡建模**：
- 收入：运营收入（床位费/月）+ 政府补贴 + 社会捐赠
- 支出：固定成本（建设折旧）+ 运营成本（人力、水电等）
- 目标：在满足需求的同时实现财务可持续

### 2020年D题：新零售目标产品的精准需求预测

**题目逻辑剖析：**

四问从宏观到微观，从分析到预测再到建议：
- **问题1**：分析节假日对目标SKC销售量的影响因素
- **问题2**：预测目标小类未来3个月月销量（给出MAPE）
- **问题3**：预测目标小类内所有SKC未来12周周销量
- **问题4**：给企业写推荐信

**建模思路产生过程：**

问题1是**相关性分析 + 回归分析**：
- 提取特征：节假日标识、折扣力度、库存水平、销售阶段（产品生命周期）
- 分析各特征与销售量的相关性（Pearson/Spearman相关系数）
- 构建多元回归模型，量化各因素影响

问题2-3是**层级时序预测（Hierarchical Forecasting）**：
- 自上而下：先预测小类总量，再分配到SKC（乘以历史分布比例）
- 自下而上：先预测每个SKC，再汇总得小类
- 最优结合：调和预测（Reconciled Forecasting）

关键挑战：SKC数量多、单品历史短（新品上市）→ 需利用**同类产品的销售曲线形态**（协同过滤或聚类后用类均值）进行迁移预测。

### 2023年D题：航空安全风险分析和飞行技术评估问题

**题目逻辑剖析：**

五问从"数据清洗 → 操纵量化 → 超限分析 → 技术评估 → 实时预警"逐层深入：
- **问题1**：数据质量可靠性研究，提取关键数据项
- **问题2**：对飞行操纵（杆位变化）进行量化描述
- **问题3**：研究超限事件的时空分布特征
- **问题4**：建立飞行技术评估模型（区分飞行员资质）
- **问题5**：设计实时自动化预警机制

**建模思路产生过程：**

问题1是**数据预处理**：
- 识别QAR数据中的异常（传感器故障、物理不可能值）
- 评估各参数与飞行安全的相关程度（信息增益/互信息）

问题2是**时序特征提取**：
- 着陆过程中的杆位变化是时间序列 → 提取关键特征：松杆时机、松杆幅度、松杆速率
- "接地前5秒下凸"对应时序的局部极小值，可用峰值检测算法（scipy.signal.find_peaks）

问题3是**关联分析**：
- 超限事件 × 机场/航线/天气/飞行员 → 建立频率矩阵
- 用卡方检验或关联规则挖掘（Apriori/FP-Growth）找显著组合

问题4是**多元评分模型**：
- 以飞行参数（G值、侧风修正量、进近稳定性等）为特征
- 目标变量：飞行员资质等级
- 可用**Fisher判别分析**或**随机森林**进行多分类

问题5是**实时异常检测**：
- 基于历史数据建立正常飞行参数的基准分布
- 实时数据到来时计算Z-score或马氏距离，触发阈值则预警
- 可用**LSTM自编码器**学习正常飞行模式，异常时重建误差激增

### 2022年A题：大规模指纹图像检索的模型与实现

**题目逻辑剖析：**

四问聚焦"检索算法设计 → 验证 → 计算 → 评估"：
- **问题1**：设计快速指纹检索方法，分析复杂度
- **问题2**：验证算法在同指集合上的穿透率
- **问题3**：对乱序数据集进行检索计算
- **问题4**：评估不同方法的优缺点及改进方向

**建模思路产生过程：**

问题的核心矛盾：精确匹配（保留所有同一对）vs 高效过滤（去除大量不同指纹）。

指纹以**细节点三元组 (x, y, θ)** 表示。检索的本质是：在库中找到与查询指纹细节点分布最相似的候选集。

检索方案设计思路：
1. **特征哈希（LSH）：** 构造对旋转/平移鲁棒的局部特征哈希，相似指纹哈希值相近
2. **分区索引：** 按细节点的方向分布（方向直方图）构建倒排索引
3. **粗筛→精筛两阶段：** 粗筛用快速哈希比较，精筛用精确几何匹配
4. **鲁棒性保证：** 由于存在弹性变形，匹配容忍一定的位置/角度偏差 → 设置宽松的匹配阈值

穿透率 = 在过滤掉x%后，仍在剩余集合中找到同一指纹的比例。

### 2021年B题：三维团簇的能量预测

**题目逻辑剖析：**

四问对金团簇和硼团簇各做"已知结构预测能量 → 搜索最优结构"的流程：
- **问题1**：Au20能量预测（监督学习回归）
- **问题2**：Au32全局最优结构搜索（优化+ML结合）
- **问题3**：B45⁻能量预测
- **问题4**：B40⁻全局最优结构搜索

**建模思路产生过程：**

关键挑战：能量是原子三维坐标的函数，但这个函数解析表达式未知 → 用机器学习从数据中学习。

特征工程是核心难点（输入坐标→特征向量）：
- 特征需满足**置换不变性**（交换两个同类原子能量不变）
- 特征需满足**旋转/平移不变性**（旋转整个团簇能量不变）
- 常用特征：原子间距离矩阵的对称函数（径向/角向分布函数，Behler-Parrinello对称函数）
- 现代方法：**图神经网络（GNN）**，将团簇建模为原子图，通过消息传递学习能量

结构搜索（问题2、4）：
- 生成异构体：通过随机扰动、遗传算法、basin-hopping等方法
- 用ML模型快速评估能量（替代DFT）→ 筛选低能量结构 → 精确计算验证

## 八、第七类：工程物理建模类

### 2021年C题：海底数据中心的散热优化设计

**题目逻辑剖析：**

五问由浅到深，从单纯散热能力估算到综合优化设计：
- **问题1**：估算圆柱形集装箱最多能放多少服务器
- **问题2**：优化外壳结构（翅片等）以最大化散热
- **问题3**：选择合适材料和深度，综合优化
- **问题4**：考虑潮汐和季节变化的影响
- **问题5**：给相关公司写建议信

**建模思路产生过程：**

问题1是**传热学建模**的核心：
- 服务器产热 = 500W/台
- 海水冷却：自然对流换热 → 圆柱外表面的对流换热系数 $h$ 由Nusselt数计算：
  $Nu = C \cdot (Gr \cdot Pr)^n$（Grashof数 × Prandtl数）
- 热平衡：总散热量 $Q = h \cdot A_{表面} \cdot \Delta T$，其中 $\Delta T = T_{服务器} - T_{海水}$
- 最多服务器数 = $Q_{max} / Q_{单台}$，同时受物理尺寸约束

问题2：增加翅片扩大换热面积，同时考虑翅片效率（翅片越长，温差越小，效率越低）→ **优化翅片几何参数**（高度、间距、数量）

问题3：更深海水温度更低（约1-4°C），散热效果更好，但需承受更大压强 → 材料选择需考虑**强度/重量/成本/耐腐蚀性**的综合权衡

## 九、第八类：定价与决策分析类

### 2020年A题：无车承运人平台线路定价问题

**题目逻辑剖析：**

四问构建"影响因素分析 → 历史定价评价 → 新线路报价 → 平台建议"框架：
- **问题1**：定量研究影响货运定价的主要因素
- **问题2**：评价历史交易数据中的定价合理性
- **问题3**：给出待定价线路的三次报价策略
- **问题4**：给平台写建议信

**建模思路产生过程：**

问题1是**特征重要性分析**：
- 潜在因素：距离、货物重量/体积、车型、出发地/目的地、时间（节假日、旺季）、竞争程度
- 方法：Pearson相关系数 + 多元回归的显著性检验 + 随机森林特征重要性

问题2是**定价评价模型**：
- 构建"合理价格"基准（线性/非线性回归预测价格）
- 实际价格 vs 预测价格的偏差 = 定价偏差指标
- 还可从**成交率**角度评价：过高则难以成交，过低则损失平台收益

问题3的**三次报价策略**是动态定价问题：
- 第一次报价：参考历史类似线路定价（基准价）
- 若未成交：第二次调价（根据剩余时间和同类线路市场情况）
- 若仍未成交：第三次调价（接近底线，确保成交）
- 可建立**马尔可夫决策过程（MDP）**模型：状态=（当前价格、剩余时间），动作=调价幅度，奖励=成交概率×收益

## 十、跨类共性规律总结

### 10.1 建模思路产生的通用框架

无论哪类题目，建模思路的产生都遵循以下路径：

```
读题 → 识别核心矛盾（优化/预测/分析？）
     → 抽象为数学对象（变量/约束/目标）
     → 选择合适模型（类比已知问题）
     → 考虑规模与可解性（精确/近似/启发式？）
     → 小规模验证 → 推广到完整问题
```

### 10.2 MathorCup赛题的典型命题规律

通过16道题的分析，可以归纳出以下命题规律：

**结构规律：**
- 几乎所有题目都分3-6个子问题，形成由简到繁的梯度
- 第一问通常是分析/基础建模，最后一问通常是开放性或拓展性问题
- 赛题往往来自真实企业场景，附有真实数据

**方法规律：**
- A题（难度最高）：强调创新性，通常涉及前沿方法（量子计算、自动驾驶等）
- B题：偏向经典运筹/优化问题的工程化应用
- C题：通常有较强的数据分析成分（预测+优化结合）
- D题：数据挖掘/评估类问题，强调统计分析能力

**评分规律：**
- 模型的合理性 > 结果的精确性
- 多目标问题需明确写出权衡逻辑
- 算法必须说明可行性和复杂度

### 10.3 高频模型与工具建议

| 应用场景 | 推荐方法 | 工具/库 |
|----------|----------|---------|
| 整数规划 | ILP/MILP | PuLP, Gurobi, CPLEX |
| 路径规划 | BFS/Dijkstra/A* | NetworkX |
| 时序预测 | SARIMA, Prophet, LSTM | statsmodels, prophet, PyTorch |
| 机器学习 | XGBoost, LightGBM | sklearn, lgbm |
| 多目标优化 | NSGA-II, 加权和 | pymoo |
| 启发式优化 | 遗传算法, 模拟退火 | scipy.optimize, deap |
| 传热分析 | Nusselt数公式 | 手算+Python验证 |
| 图算法 | DFS/BFS/并查集 | NetworkX |
